The graphics below explore some variables in the dataset. The three we are interested in gaining information from are: Author, Quote, and Genre.

Count of Quotes per Genre

This histogram shows the number of quotes in each of the 117 genres. The most common genre is age and is closely followed by alone, art, business and amazing. There are close to 1,000 quotes labeled with each of these genres. The genres with the fewest quotes are Mother's Day, St.Patrick's Day, Father's Day, Memorial Day, and Valentine's Day. These genres are very close to the x-axis, so their counts are close to zero. The counts of these quotes are all less than 20 each.

Count of Quotes per Author

This histogram shows the number of quotes per author. Since there are 11,173 unique authors this histogram only shows the top 1,000 authors. It is clear from this graph that most of these authors write approximately 60 quotes each or less, but the maximum number of quotes an author wrote is just below 140.

Number of Words vs. Characters per Quote

This scatterplot shows the relationship between the number of words per quote compared to the number of characters per quote. The plot clearly shows that as the number of words increase, the number of characters increase and as the number of characters increase, the number of words increase.

Number of Quotes with <=280 Characters

This histogram shows whether or not the quote is 280 characters or less. If there are 280 characters or less, then the quote is labeled True, but if there are more than 280 characters, the quote is labeled False. The number of characters in a quote is important when we post them to twitter because a tweet can only have a maximum length of 280 characters.

Machine Learning Validation Comparison

After analyzing each of the models’ loss vs. validation loss plots, we decided the best model was the Stacked LSTM model. We chose this model because all of the other models’ losses and validation losses diverged from each other very quickly. While the Stacked LSTM did not have very good values for loss, we thought it was the best choice because the training loss and validation loss were close to each other until 6 epochs. Due to the divergence of the two after 6 epochs, we decided to only train the model for 6 epochs. Therefore the quote on the Quote Generator page will be generated using the Stacked LSTM model.

Number of Words in Each Quote: Original Dataset vs. Twitter Data

The graphs above show the word counts in quotes. The histogram to the left represents the word counts of quotes from the original dataset and the histogram to the right represents the word counts of quotes from the data pulled from various Twitter accounts. It is clear looking at these histograms that most of the data is currently being pulled from the original dataset. The histograms also show that some of the quotes from the dataset are 20 to 40 words longer than the quotes pulled from Twitter. The quote with the maximum length from the original dataset is over 80 words, and the quote with the maximum length from the Twitter data, is about 47 words. However, the peak of both curves is around the 20 word mark.

Number of Characters in Each Quote: Original Dataset vs. Twitter Data

The visualizations above represent the number of characters in each quote in the original dataset (histogram on left) and the number of characters in each quote in the Twitter Data (histogram to right). The maximum character count in a tweet is 280, so it makes sense that the maximum number of characters of the Twitter data is about 250. The original dataset has character counts that are more than 280. The maximum character count is over 400 characters. The peak of both histograms are at about 100 characters. In the data from Twitter most of the quotes are between the 50 and 150 characters range, but in the original dataset, a lot of the data falls above this range, mostly between the 200 and 300 characters count.
